  <action>
    **Fix cookie security per D-10 and CONCERNS.md (JWTs are accepted through query parameters and stored in browser storage):**

    1. In `backend/src/axum_routers/auth.rs`:
       - Change `same_site(SameSite::None)` to `same_site(SameSite::Strict)` on ALL cookie builds
       - Add `.secure(true)` to ALL cookie builds (requires HTTPS, prevents MITM)
       - Remove support for `token=` query parameter login (`post_login_with_token` should only accept body credentials)
       - The JWT should ONLY be settable via httpOnly cookie, not readable by JavaScript

    2. In `backend/src/hoops/jwt.rs`:
       - Update `auth_middleware` to extract JWT from httpOnly cookie via `CookieJar` extractor instead of Authorization header
       - Keep Authorization header support as fallback for API clients (Brave, curl), but prioritize cookie
       - Extract cookie using `jar.get("jwt_token")` when CookieJar is available

    3. In `backend/src/hoops/jwt.rs`:
       - Add `CookieJar` as first extractor in auth_middleware to check for httpOnly cookie
       - Fall back to Authorization Bearer header if cookie not present
       - Reject tokens that appear in query strings (remove TokenQuery support from login)

    **D-10 requirement:** JWT moved from localStorage to httpOnly cookies with Secure and SameSite=Strict flags.
    **CONCERNS.md:** Remove token-in-query support, stop persisting JWTs in localStorage.
  </action>
  <verify>
    <automated>cd backend && cargo check --lib</automated>
  </verify>
  <done>All login endpoints set httpOnly+Secure+SameSite=Strict cookies. Token query parameter login removed. Auth middleware extracts from cookie first, header second.</done>
