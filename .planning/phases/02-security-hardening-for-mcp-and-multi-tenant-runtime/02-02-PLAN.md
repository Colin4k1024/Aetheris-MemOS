    2. Create `backend/src/mcp/signing.rs` with:

       ```rust
       use sha2::{Sha256, Digest};

       /// Component signature containing SHA-256 hash and issuer metadata
       #[derive(Debug, Clone, Serialize, Deserialize)]
       pub struct ComponentSignature {
           pub component_id: String,
           pub sha256_hash: String,           // hex-encoded SHA-256 of component artifact
           pub issuer: String,                // issuer identifier (e.g., "adaptive-memory-system")
           pub version: String,                // component version
           pub timestamp: i64,               // signature creation timestamp
           pub signature: String,             // actual signature bytes (hex-encoded)
       }

       /// Signing verification error
       #[derive(Debug, thiserror::Error)]
       pub enum SigningError {
           #[error("Component {0} is not signed")]
           Unsigned(String),
           #[error("Signature verification failed for {0}: {1}")]
           VerificationFailed(String, String),
           #[error("Trusted key bundle not found")]
           MissingKeyBundle,
       }

       /// Trusted key bundle for signature verification
       pub struct TrustedKeyBundle {
           keys: HashMap<String, Vec<u8>>,  // issuer -> public key bytes
       }

       /// Verify component signature against trusted key bundle
       pub fn verify_component(
           component_id: &str,
           artifact: &[u8],
           signature: &ComponentSignature,
           key_bundle: &TrustedKeyBundle,
       ) -> Result<(), SigningError> {
           // 1. Compute SHA-256 of artifact
           // 2. Compare with signature.sha256_hash (constant-time comparison)
           // 3. Verify signature using issuer's public key
           // 4. Return error if any check fails
       }
       ```

    3. For MVP, implement HMAC-SHA256 based signing:
       - Key bundle contains shared secrets per issuer (loaded from environment)
       - `signature` = HMAC-SHA256(key, sha256_hash + issuer + version + timestamp)
       - This avoids full PKI complexity while providing integrity guarantees

    4. Support a `MCP_TRUSTED_ISSUERS` environment variable:
       - JSON array of trusted issuer IDs
       - Corresponding keys loaded from `MCP_KEY_{ISSUER}` env vars

    **D-01:** MCP components must be signed; verify signature on load using a trusted key bundle.
    **D-02:** Provenance checks use SHA-256 hash of component artifact + issuer metadata.
    **D-03:** Unverified or unsigned components are rejected at load time with a structured error.

    2. Create a `McpComponentRegistry` state:
       ```rust
       #[derive(Clone)]
       pub struct McpComponentRegistry {
           key_bundle: Arc<TrustedKeyBundle>,
           verified_tools: RwLock<HashMap<String, Tool>>,
           verified_resources: RwLock<HashMap<String, Resource>>,
       }

       impl McpComponentRegistry {
           pub fn new() -> Self {
               let key_bundle = TrustedKeyBundle::load_from_env();
               Self {
                   key_bundle: Arc::new(key_bundle),
                   verified_tools: RwLock::new(HashMap::new()),
                   verified_resources: RwLock::new(HashMap::new()),
               }
           }

           pub fn register_tool(&self, tool: Tool, signature: ComponentSignature) -> Result<(), SigningError> {
               verify_component(&tool.name, tool.artifact_bytes(), &signature, &self.key_bundle)?;
               self.verified_tools.write().unwrap().insert(tool.name.clone(), tool);
               Ok(())
           }
       }
       ```

    3. For each tool in `get_memory_tools()`:
       - Tool definition includes a `signature` field
       - If signature missing or invalid, tool is NOT included in returned list
       - Log warning: "MCP tool {} rejected: signing verification failed"

       - Include error code, component_id, issuer, and timestamp in the error response
       - This helps security auditors identify which components failed verification

    **D-03:** Unverified or unsigned components are rejected at load time with a structured error.
