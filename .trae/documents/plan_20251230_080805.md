# 后端代码优化与 todos 功能实现计划

## 1. 项目分析与架构设计

### 1.1 现有架构评估

* **框架**: Rust + Salvo

* **数据库**: SQLite (sqlx)

* **认证**: JWT

* **日志**: Tracing

* **API 文档**: Scalar/OpenAPI

### 1.2 todos 功能设计

#### 1.2.1 数据模型

```rust
pub struct Todo {
    id: String,
    title: String,
    description: Option<String>,
    completed: bool,
    created_at: String,
    updated_at: String,
    user_id: String,
}
```

#### 1.2.2 数据库表设计

```sql
CREATE TABLE todos (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    description TEXT,
    completed BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    user_id TEXT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users (id)
);
```

#### 1.2.3 API 设计

| 方法     | 路径             | 描述         | 认证 |
| ------ | -------------- | ---------- | -- |
| GET    | /api/todos     | 获取所有 todos | 需要 |
| GET    | /api/todos/:id | 获取单个 todo  | 需要 |
| POST   | /api/todos     | 创建新 todo   | 需要 |
| PUT    | /api/todos/:id | 更新 todo    | 需要 |
| DELETE | /api/todos/:id | 删除 todo    | 需要 |
