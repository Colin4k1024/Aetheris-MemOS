# 前端冗余页面移除计划

## 1. 识别冗余页面

根据对前端项目的分析，以下页面是从 Ant Design Pro 模板遗留下来的，与自适应记忆系统的业务无关，应被移除：

- `Admin.tsx`：模板遗留的管理员页面，显示 Ant Design Pro 欢迎信息
- `Welcome.tsx`：模板遗留的欢迎页面，介绍 Ant Design Pro 功能
- `table-list/`：示例表格页面，使用模拟数据和 API

## 2. 移除步骤

### 步骤 1：移除路由配置
从 `config/routes.ts` 中移除以下路由配置：
- `/admin` 路由及其子路由
- `/welcome` 路由
- `/list` 路由

### 步骤 2：移除页面文件
删除以下文件和目录：
- `src/pages/Admin.tsx`
- `src/pages/Welcome.tsx`
- `src/pages/table-list/` 目录及其所有子文件

### 步骤 3：清理国际化配置
检查并移除 `src/locales/` 目录下与这些页面相关的国际化配置：
- 搜索包含 "admin"、"welcome"、"table-list" 或 "list.table-list" 的国际化配置
- 删除相关配置项

### 步骤 4：清理服务调用
检查 `src/services/` 目录下的 API 服务，移除与 table-list 相关的模拟 API：
- 检查 `src/services/ant-design-pro/api.ts` 中的相关服务
- 移除 `removeRule` 和 `rule` 等与 table-list 相关的 API 调用

### 步骤 5：清理组件引用
检查其他组件是否引用了要移除的页面或组件：
- 使用 `grep` 命令搜索项目中对这些页面的引用
- 移除或更新相关引用

### 步骤 6：功能测试和构建验证
- 运行 `npm run build` 验证项目是否能成功构建
- 运行 `npm run dev` 启动开发服务器，测试现有功能是否正常

## 3. 预期效果

- 减少项目代码量，提高代码可维护性
- 移除无关的示例代码，使项目结构更清晰
- 降低后续开发和维护成本
- 确保系统功能不受影响

## 4. 注意事项

- 在移除页面之前，确保这些页面确实没有被其他组件或功能引用
- 移除路由配置时，确保没有其他路由依赖这些路由
- 在修改国际化配置时，确保没有其他功能使用了相同的配置键
- 测试过程中，重点关注与记忆系统相关的核心功能是否正常工作
