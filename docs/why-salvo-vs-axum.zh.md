# 为什么选择 Salvo（未来可选 Axum）

HTTP 层的技术选择及其如何适配开源路线图。

---

## 当前选择：Salvo

后端使用 **Salvo 0.84** (Rust) 进行路由、提取、OpenAPI 和中间件。

- **选择 Salvo 的理由：** 成熟的 Rust 异步 Web 框架；内置 OpenAPI 支持；直接的处理器和提取器模型。项目从 Salvo 开始，代码库与其保持一致。
- **权衡：** Salvo 的生态系统比 **Axum** 小（Axum 的教程、示例和第三方集成更丰富）。对于开源项目，"为什么不选 Axum？" 是个合理的问题。

---

## 生态系统对比（简要）

| 方面 | Salvo | Axum |
| ---- | ----- | ---- |
| 社区规模 | 较小 | 事实标准 |
| 教程/示例 | 较少 | 很多 |
| Agent/AI 集成示例 | 罕见 | 越来越多 |
| 企业采用可见度 | 较低 | 较高 |

本项目的**逻辑和设计**（自适应记忆、策略、代理）与框架无关。HTTP 层是服务和数据库的薄包装。

---

## 路线图：可选的 Axum 适配器

我们**不会**默认从 Salvo 迁移。为了提高开源可信度和与 Rust Web 生态系统的契合度：

- **v0.4** 可能引入**可选的 Axum 后端适配器**：相同的 API 和服务层，不同的路由和服务器入口点。Salvo 保持默认；Axum 成为标准化使用它的团队的替代方案。

这使核心价值（自适应记忆、代理抽象、策略）独立于 Web 框架。

请参阅 [ROADMAP.md](ROADMAP.md) 了解版本规划。
