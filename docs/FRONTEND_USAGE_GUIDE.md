# 自适应内存系统前端使用指南

## 1. 前端架构

### 1.1 技术栈

- **框架**: React 18
- **UI组件库**: Ant Design Pro 6.0
- **开发框架**: Umi 4
- **状态管理**: React Hooks
- **API请求**: Axios
- **样式方案**: Less + CSS Modules
- **图表库**: @ant-design/charts

### 1.2 项目结构

前端项目位于 `frontend/ant-design-pro-template` 目录下，主要结构如下：

```
frontend/ant-design-pro-template/
├── config/            # 项目配置 (路由、代理等)
├── mock/              # 模拟数据
├── public/            # 静态资源
├── src/
│   ├── components/    # 公共组件
│   ├── locales/       # 国际化配置
│   ├── pages/         # 页面组件
│   │   ├── Dashboard/ # 仪表盘
│   │   ├── TaskAnalysis/ # 任务分析
│   │   ├── MemoryConfig/ # 记忆配置
│   │   ├── MemoryDecisionTrace/ # 决策追踪
│   │   ├── MemoryManagement/ # 记忆管理
│   │   ├── Performance/ # 性能监控
│   │   ├── ResourceMonitor/ # 资源监控
│   │   └── WeightHistory/ # 权重历史
│   ├── services/      # API服务
│   ├── app.tsx        # 应用入口
│   └── global.less    # 全局样式
├── tests/             # 测试文件
└── types/             # TypeScript类型定义
```

## 2. 页面导航

### 2.1 导航结构

系统采用左侧导航栏 + 顶部导航栏的布局方式：

- **顶部导航栏**: 显示当前用户信息、系统通知、语言切换等
- **左侧导航栏**: 包含系统主要功能模块，具体如下：

| 导航项 | 路径 | 功能 |
|--------|------|------|
| 仪表板 | /dashboard | 系统概览、关键指标展示 |
| 任务分析 | /task-analysis | 任务特征分析、记忆策略推荐 |
| 记忆配置 | /memory-config | 记忆配置管理、权重调整 |
| 决策追踪 | /memory-decision-trace | 决策链路追踪、回溯分析 |
| 记忆管理 | /memory-management | 记忆存储与检索管理 |
| 性能监控 | /performance | 系统性能监控、预测模型评估 |
| 资源监控 | /resource-monitor | 系统资源使用情况监控 |
| 权重历史 | /weight-history | 记忆权重调整历史记录 |

### 2.2 登录页面

**路径**: `/user/login`

**功能**: 用户登录系统，获取认证凭证

**使用方法**:
1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后自动跳转到仪表板页面

**默认账号**（仅用于本地/演示环境，生产环境请修改）:
- 用户名: 参见部署时创建的账号
- 密码: 参见部署时设置的密码

## 3. 页面功能说明

### 3.1 仪表板

**路径**: `/dashboard`

**功能**: 展示系统概览和关键指标，提供快速访问入口

**主要内容**:

1. **系统状态卡片**: 显示系统健康状态、响应时间、成功率等关键指标
2. **资源使用图表**: 实时展示CPU、内存、存储等资源使用情况
3. **记忆配置概览**: 展示当前记忆配置和权重分布
4. **最近任务分析**: 显示最近的任务特征分析结果
5. **快速操作**: 提供常用功能的快速访问按钮

**使用方法**:
1. 查看系统状态卡片，了解系统整体运行情况
2. 观察资源使用图表，监控系统资源消耗
3. 点击记忆配置概览，进入记忆配置页面
4. 点击最近任务分析，查看详细的任务特征分析
5. 使用快速操作按钮，快速访问常用功能

### 3.2 任务分析

**路径**: `/task-analysis`

**功能**: 分析任务特征，推荐最优记忆策略

**主要内容**:

1. **任务特征输入**: 提供任务内容、模态要求等输入界面
2. **特征分析结果**: 展示任务复杂度、推理深度等分析结果
3. **记忆策略推荐**: 根据任务特征推荐最优记忆配置
4. **历史分析记录**: 显示历史任务分析记录

**使用方法**:
1. 在任务特征输入区域，填写任务内容和相关信息
2. 点击"分析任务特征"按钮，系统开始分析
3. 查看特征分析结果，了解任务的关键特征
4. 查看推荐的记忆策略，包括主要记忆类型和权重
5. 点击"应用配置"按钮，将推荐配置应用到系统
6. 在历史分析记录中查看之前的分析结果

### 3.3 记忆配置

**路径**: `/memory-config`

**功能**: 管理系统记忆配置，调整记忆权重

**主要内容**:

1. **记忆配置列表**: 展示所有可用的记忆配置
2. **配置详情**: 显示当前配置的详细信息
3. **权重调整**: 提供可视化的权重调整界面
4. **配置管理**: 提供创建、编辑、删除配置的功能

**使用方法**:
1. 在记忆配置列表中选择一个配置
2. 查看配置详情，了解配置的基本信息
3. 在权重调整区域，使用滑块或输入框调整各记忆层的权重
4. 点击"保存配置"按钮，保存调整后的配置
5. 点击"创建配置"按钮，创建新的记忆配置
6. 点击"删除配置"按钮，删除不需要的配置

### 3.4 决策追踪

**路径**: `/memory-decision-trace`

**功能**: 展示记忆选择的完整决策链路，支持决策回溯和分析

**主要内容**:

1. **决策列表**: 显示所有记忆选择决策记录
2. **决策详情**: 展示决策的完整链路（分析器→预测器→权重调整→调度器）
3. **输入输出**: 显示决策的输入参数和最终结果
4. **时间线**: 按时间顺序展示决策历史

**使用方法**:
1. 在决策列表中查看所有决策记录
2. 点击某条决策，查看详细的决策链路
3. 分析决策过程，了解每个环节的影响
4. 使用时间线功能，追溯历史决策

### 3.5 性能监控

**路径**: `/performance`

**功能**: 监控系统性能，评估预测模型准确性

**主要内容**:

1. **性能指标图表**: 展示效率增益、连贯性增益等指标
2. **预测模型评估**: 显示预测模型的准确性评估
3. **性能趋势分析**: 展示性能指标的历史趋势
4. **配置性能对比**: 对比不同配置的性能表现

**使用方法**:
1. 查看性能指标图表，了解系统当前性能
2. 观察预测模型评估结果，评估模型准确性
3. 分析性能趋势，了解系统性能变化
4. 使用配置性能对比功能，比较不同配置的性能差异
5. 根据性能数据，调整系统配置

### 3.6 资源监控

**路径**: `/resource-monitor`

**功能**: 实时监控系统资源使用情况，提供资源优化建议

**主要内容**:

1. **资源使用实时监控**: 实时展示CPU、内存、网络等资源使用情况
2. **资源历史趋势**: 展示资源使用的历史趋势
3. **资源预警设置**: 设置资源使用阈值，触发预警
4. **资源优化建议**: 根据资源使用情况提供优化建议

**使用方法**:
1. 查看资源使用实时监控，了解系统当前资源消耗
2. 观察资源历史趋势，分析资源使用规律
3. 设置资源预警阈值，当资源使用超过阈值时接收预警
4. 查看资源优化建议，根据建议优化系统配置
5. 点击"生成优化报告"按钮，生成详细的资源优化报告

### 3.6 权重历史

**路径**: `/weight-history`

**功能**: 记录和展示记忆权重调整历史，分析调整效果

2. **调整详情**: 显示每次调整的详细信息
3. **调整效果分析**: 分析权重调整对系统性能的影响
4. **调整趋势**: 展示权重调整的历史趋势

**使用方法**:
1. 在权重调整记录列表中查看所有调整记录
2. 点击某条记录，查看详细的调整信息
3. 分析调整效果，了解权重调整对系统性能的影响
4. 观察调整趋势，分析系统配置的演变
5. 使用筛选功能，根据时间、调整类型等筛选记录

### 3.7 记忆管理

**路径**: `/memory-management`

**功能**: 管理系统记忆存储，提供记忆检索功能

**主要内容**:

3. **记忆详情**: 展示记忆条目的详细信息
4. **记忆管理**: 提供记忆条目管理功能

**使用方法**:
1. 查看记忆存储概览，了解不同类型记忆的存储情况
2. 在记忆检索区域，选择检索方式
3. 输入检索条件，点击"搜索"按钮
4. 查看搜索结果，了解相关记忆条目
5. 点击某条记忆，查看详细信息
6. 使用管理功能，对记忆条目进行编辑、删除等操作

## 4. 操作流程

### 4.1 基本操作流程

1. **登录系统**: 访问登录页面，输入账号密码登录
2. **查看仪表板**: 登录后自动进入仪表板，查看系统概览
3. **分析任务**: 进入任务分析页面，分析任务特征
4. **配置记忆**: 根据任务分析结果，进入记忆配置页面调整配置
5. **执行任务**: 使用调整后的配置执行任务
6. **监控性能**: 进入性能监控页面，监控系统性能
7. **优化资源**: 根据性能监控结果，进入资源监控页面优化资源

### 4.2 记忆配置调整流程

1. **进入记忆配置页面**: 从左侧导航栏点击"记忆配置"
2. **选择配置**: 在配置列表中选择一个配置
3. **调整权重**: 使用滑块或输入框调整各记忆层的权重
4. **保存配置**: 点击"保存配置"按钮保存调整
5. **应用配置**: 点击"应用配置"按钮，将配置应用到系统
6. **监控效果**: 进入性能监控页面，监控配置调整效果

### 4.3 记忆检索流程

1. **进入记忆管理页面**: 从左侧导航栏点击"记忆管理"
2. **选择检索方式**: 选择关键词检索、语义检索或混合检索
3. **输入检索条件**: 根据选择的检索方式，输入相应的检索条件
4. **执行检索**: 点击"搜索"按钮执行检索
5. **查看结果**: 查看检索结果列表
6. **查看详情**: 点击某条结果，查看记忆条目的详细信息
7. **相关操作**: 对记忆条目进行编辑、删除等操作

## 6. 常见问题

### 6.1 页面加载缓慢

- 检查网络连接是否正常
- 检查后端服务是否正常运行
- 清除浏览器缓存后重新加载

### 6.2 登录失败

- 检查用户名和密码是否正确
- 检查后端服务是否正常运行
- 检查网络连接是否正常

### 6.3 功能无法使用

- 检查是否有操作权限
- 检查后端服务是否正常运行
- 查看浏览器控制台是否有错误信息

### 6.4 数据显示异常

- 检查后端服务是否正常运行
- 检查数据库连接是否正常
- 清除浏览器缓存后重新加载

### 6.5 系统预警

- 查看预警详情，了解预警原因
- 根据预警建议，调整系统配置
- 如无法解决，联系系统管理员

## 9. 开发与定制

### 9.1 本地开发

1. 进入前端目录：
   ```bash
   cd frontend/ant-design-pro-template
   ```

2. 安装依赖：
   ```bash
   npm install
   ```

3. 启动开发服务器：
   ```bash
   npm run dev
   ```

4. 访问开发地址：
   ```
   http://localhost:8000
   ```

### 9.2 构建生产版本

1. 进入前端目录：
   ```bash
   cd frontend/ant-design-pro-template
   ```

2. 构建生产版本：
   ```bash
   npm run build
   ```

3. 构建结果位于 `dist` 目录下

### 9.3 定制开发

- **修改主题**: 在 `src/global.less` 中修改全局样式
- **添加页面**: 在 `src/pages` 目录下创建新的页面组件
- **修改导航**: 在 `config/routes.ts` 中修改导航配置
- **添加API**: 在 `src/services` 目录下添加新的API服务
