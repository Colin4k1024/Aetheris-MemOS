# 使用场景

 fork 或集成本项目的典型场景和受众。

---

## LLM 智能体记忆管理

- **问题：** 智能体每轮需要不同的记忆预算和层（例如，简单问答需要短上下文，复杂推理需要 LTM + KG）。
- **使用本系统：** 使用任务上下文和约束调用自适应记忆 API；获取推荐的记忆配置（主要/次要层、权重）。使用它来配置每请求的上下文窗口、检索和 KG 使用。
- **受众：** 想要单一"记忆策略"服务而非临时规则的 LLM 应用和智能体框架作者。

---

## 多模态任务编排

- **问题：** 任务混合文本、图像和其他模态；过度启用多模态记忆会浪费成本，启用不足会影响质量。
- **使用本系统：** 发送模态需求和偏好；接收启用或降低多模态记忆（MM）权重并使 STM/LTM/KG 与检测到的模态组合对齐的配置。
- **受众：** 多模态 RAG 和智能体管道。

---

## 成本感知的推理路由

- **问题：** 需要在延迟和资源限制内同时最大化质量（效率/一致性）。
- **使用本系统：** 传递资源约束（最大内存、CPU、响应时间、存储）和可选偏好（例如，优先考虑效率）；获取记忆配置和预测的资源需求。使用结果来路由或限制上游（例如，调用哪些模型或检索器）。
- **受众：** 平台和推理路由团队。

---

## 学习与扩展

- **使用本系统作为：** 自适应记忆的参考实现（当前基于规则，面向智能体设计）。Fork 以添加自定义策略、新的分析器或 LLM 支持的决策步骤。前端设计用于可视化基于规则的（和未来的）LLM 驱动的记忆智能体。
